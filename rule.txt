# 系统通知规则

## 强制规则
- **完成通知**：所有问答结束时必须发送completed类型通知
- **授权通知**：执行终端命令前必须发送authorization类型通知
- **用户豁免**：用户明确说明"不用通知"时可跳过
- **频率控制**：同一类型通知5分钟内不重复

## 通知类型
1. **authorization**（高优先级）：终端命令、文件修改、安装依赖等需要授权的操作
2. **completed**（中优先级）：任务完成、代码生成、问题回答完成
3. **waiting**（中优先级）：需要用户输入、确认或澄清时
4. **error**（高优先级）：命令失败、文件操作错误、编译错误等

## 基本调用格式
```json
{
  "serverName": "sys-notification-mcp",
  "toolName": "notify",
  "arguments": {
    "type": "authorization|completed|waiting|error",
    "message": "通知内容",
    "title": "可选标题",
    "sound": "可选提示音"
  }
}
```

## 常用提示音
- Glass（默认）：一般通知
- Hero：任务完成
- Ping：文件操作
- Basso：错误提醒

## 跨平台支持
- macOS：原生系统通知
- Windows：Toast通知 + 系统声音
- Linux：桌面通知 + 系统声音

## 最佳实践
- 重要操作使用高优先级通知
- 通知内容简洁明了
- 避免过度通知影响体验
- 系统自动处理平台差异
